## Key Technical Decisions

- **`ON DUPLICATE KEY UPDATE` in `sp_upsert_rating`**: eliminates the need for a separate "does this rating exist?" check in the service layer; the database handles the upsert atomically
- **Automatic watch history on rating**: combining rating insert and history insert in a single stored procedure ensures the two tables are always consistent — it is impossible to rate a show you have not "watched"
- **Pre-aggregated views**: `vw_shows_with_ratings` means the catalogue query never performs a GROUP BY over the full `ratings` table at query time; the view's aggregation is computed once and reused
- **`INSERT IGNORE` in `sp_mark_watched`**: watching the same show twice does not raise an error and does not update `watched_at` — the first watch date is preserved as the canonical record
